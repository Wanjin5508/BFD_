Heading Exercise

Create a blank document using a word-processor of your choice

Tasks:

  1. Include “This is a heading 1” as a first level heading. 
  2. Include “This is a heading 2” as a second level heading.
  3. Include “This is a heading 3” as a level 3 heading.
  4. Include “This is a heading 4” as a level 4 heading. 
  5. Include “This is a paragraph. Lorem ipsum…” as Body text. ( Note in some word processors this is referred to as Normal text).

In each of the above tasks when you applied the appropriate heading style, a heading structural element was created. In the following tasks you are asked to mimic each of the headings that were created in tasks 1 to 4 just using text formatting. The difference will be that the output will simply be formatted text that looks like a heading. Assistive technology screen readers will not be able to use it for navigation and document skimming and other useful processes.

Tasks:

  1. Mimic a heading 1 by simply applying appropriate text formatting to “This is a heading 1”. It should look the same as a heading 1. In other words it should be bolded if necessary , use a  similar font, and have a similar font size and other similar features to that of the heading but it should not be designated as a heading.. 
  2. Mimic a heading 2 by simply applying appropriate  text formatting to “This is a heading 2”. It should look the same as a heading 2. 
  3. Mimic a heading 3 by simply applying appropriate  text formatting to “This is a heading 3”. It should look the same as a heading 3. 
  4. Mimic a heading 4 by simply applying appropriate  text formatting to “This is a heading 4”. It should look the same as a heading 4. 
  5. Change the style on Body Text so that the font is a sans serif font like Verdana or Arial and the font size is 11-point and the text is left justified. Serif based fonts and justified text can present difficulties for those with print difficulties.

In Tasks 6 to 10 you have managed to create formatted text that is visually similar to properly defined headings. However assistive technology software (like screen readers) will not recognize it as a heading. It will process it as standard text. The user will not be aware that these are meant to be headings and will not be able to use them for navigational purposes, which is a key accessibility factor.  For example, for the same purpose that sighted readers quickly scan a document, screen readers list headings so users can get a quick overview of a document’s contents. Therefore, it is very important that headings are used to structure documents.
